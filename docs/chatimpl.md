# 项目需求文档：实现 Chat 对话功能

## 1. 功能概述
该功能旨在实现一个可持续对话的聊天系统，支持动态调用工具（如 FunctionCall），并且可以显示调用事件。同时，所有对话和工具调用数据需要持久化存储在数据库中，前端通过 SSE 实时展示这些事件。

## 2. 需求详细说明

### 2.1 对话持久化
- **功能描述**：每次与用户的对话都需要持久化存储在数据库中，以便后续查询和分析。对话数据应包括：
    - 用户输入
    - 系统回复
    - 工具调用事件
    - 会话ID、时间戳等附加信息
- **技术要求**：
    - 使用现有的数据库集成代码（如 MySQL）进行对话数据的存储。
    - 存储数据时需支持按会话ID进行索引和查询，确保数据的可检索性和可扩展性。

### 2.2 工具调用（FunctionCall）
- **功能描述**：系统需要支持在对话过程中调用外部工具（如 Service SPI）。工具调用可以转化为 `tool` 或 `skill`，并传入模型进行处理。
- **技术要求**：
    - **FunctionCall**：使用 `FunctionCall` 进行工具调用，并且通过 SPI 的 Service 工具传递工具或技能。
    - **动态工具调用**：根据用户输入和当前对话的上下文，动态选择并调用合适的工具。
    - **调用事件记录**：每次工具调用的参数、返回值等信息需要存储在数据库中，并通过 SSE 推送至前端。

### 2.3 SSE 事件推送
- **功能描述**：在对话过程中，后端将通过 SSE (Server-Sent Events) 向前端推送对话信息及工具调用事件。
- **技术要求**：
    - 后端通过 SSE 向前端推送每个工具调用的事件，包括调用参数、返回值等信息。
    - 前端能够实时显示这些事件，以便用户看到系统与外部工具交互的过程。

### 2.4 前端展示（Vue 3）
- **功能描述**：前端使用 Vue 3 接收 SSE 事件并实时展示工具调用事件及对话信息。
- **技术要求**：
    - 前端通过 SSE 与后端进行通信，接收并展示工具调用事件。
    - 确保 UI 能够在接收到事件后即时刷新，显示每次工具调用及其结果。

## 3. 架构设计
- **后端框架**：使用 LangChain4J 进行对话上下文管理和工具调用，结合 Spring Boot 或其他 Java 后端框架进行服务搭建。
- **前端框架**：使用 Vue 3 接收 SSE 事件并展示工具调用及对话内容。
- **数据持久化**：使用 MySQL 或其他关系型数据库存储对话数据和工具调用事件。

## 4. 接口设计

### 用户输入接口
- **请求方式**：HTTP POST 请求
- **请求体**：包含用户输入及会话ID（如果有）
- **返回**：系统生成的响应、工具调用事件数据、更新后的对话上下文

### 工具调用接口
- **请求**：根据用户输入动态选择合适的工具或技能进行调用（使用 `FunctionCall`）。
- **返回**：工具调用的结果及相关事件信息。

### SSE 接口
- 后端通过 SSE 向前端推送每个工具调用的事件，包括调用参数、返回值等。
- 事件内容：包括工具调用的参数、返回结果等。

## 5. 技术选型

- **LangChain4J**：用于管理对话上下文和工具调用，开发文档地址:https://docs.langchain4j.dev/intro。
- **Spring Boot**：用于构建后端服务，处理 SSE 事件和工具调用。
- **数据库**：MySQL 或其他关系型数据库用于存储对话数据和工具调用事件。
- **前端**：Vue 3 用于实现与 SSE 的集成，实时显示对话和工具调用事件。

## 6. 开发步骤

1. **搭建后端框架**：使用 Spring Boot 搭建后端，集成 LangChain4J 进行对话管理。
2. **集成 SPI 服务工具**：根据需要将 SPI 的 Service 工具转换为 tool 或 skill，并与 LangChain4J 配合使用。
3. **实现工具调用（FunctionCall）**：使用 `FunctionCall` 接口进行动态工具调用，并将调用结果持久化到数据库。
4. **实现 SSE 推送功能**：
    - 后端通过 SSE 向前端推送对话和工具调用事件。
    - 确保每个事件都包括工具调用的参数、返回结果等信息。
5. **前端实现 Vue 3 与 SSE 集成**：
    - 前端通过 SSE 接收实时事件，并在 UI 上实时显示工具调用事件。
    - 更新对话内容和工具调用过程，确保用户能够实时看到交互的结果。

## 7. 系统拓展
- 后期可以根据需求添加更多的工具集成，优化对话模型，提升工具调用的灵活性。
- 支持不同类型的事件推送和 UI 展示方式，增加系统的可扩展性和灵活性。
